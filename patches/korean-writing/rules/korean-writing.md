# 한국어 글쓰기 규칙

한국어로 **문서나 산문 산출물**을 만들 때 적용한다 — 계획 윤문본(`.readable.md`), README.ko,
리포트, 설명 문서, 긴 한글 안내 등. 짧은 대화형 답변에는 적용하지 않는다(과해진다).

## 2단계: 예방 → 치료

1. **쓰면서 예방** — `writing-tropes.md`의 회피 규칙을 따른다. 특히 번역투(그/그녀 직역,
   가지고 있다 같은 light verb, 연결어미 뒤 쉼표, "X에서 Y로" 변환공식, 의인화 추상주어)를
   처음부터 피한다.
2. **쓴 뒤 치료(윤문 패스)** — 초안을 끝낸 뒤 사람에게 보여주기 전에 humanize-korean으로
   윤문 + 검증 패스를 한 번 거친다.
   - `/humanize <파일경로|텍스트>`를 호출한다. 호출이 어려운 환경이면 humanize-korean의
     quick-rules 처방과 아래 자체검증 6항을 직접 적용한다.
   - 짧은 글은 Fast 모드로 충분하다. 길거나(8천 자+) 정확도가 중요하면 `--strict`.

## 의미 보존 (절대 규칙)

윤문은 **표현만** 바꾼다. 다음은 절대 바꾸지 않는다: 고유명사, 수치, 직접 인용, 코드·명령,
법·규정 문구, 학술 개념어. 원문에 없던 비유나 수사를 지어내지 않는다. 변경률이 과도하면
(대략 30% 이상) 과윤문이니 되돌려 다시 한다.

## 자체검증 6항 (윤문 직후 빠른 점검)

1. 고유명사·수치·인용이 그대로 보존됐나
2. 변경이 과도(30%+)하지 않나
3. 장르를 벗어나지 않았나 (리포트는 리포트답게)
4. 격식체(존댓말/평어)가 일관적인가
5. 결정적 AI 티가 남아 있지 않나
6. 원문에 없던 인공적 수사를 더하지 않았나

> 역할 분담: `writing-tropes.md` = 예방 규칙의 단일 기준. humanize-korean = 치료 + 검증
> 파이프라인. 둘은 상보적이다 — 예방은 우리 규칙, 윤문·검증은 플러그인.
